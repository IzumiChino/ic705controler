### 3. 修复地面站名称重复导致崩溃问题

**问题**: 用户报告地面站设置存在严重 Bug：
1. 保存地面站命名为 "test"
2. 返回主界面，点击位置重新获取 GPS
3. 再次进入地面站设置，命名为 "test"
4. 应用闪退或崩溃

**根本原因分析**:
- 数据库表 `stations` 的 `name` 字段有 `UNIQUE` 约束
- `insertStation` 方法使用了 `CONFLICT_REPLACE` 策略
- 当插入同名地面站时会删除旧记录并插入新记录，导致 ID 变化
- 当用户保存同名地面站时触发冲突替换，造成数据不一致

**解决方案**:

**修改文件 1**: `app/src/main/java/com/bh6aap/ic705Cter/data/database/DatabaseHelper.kt`
- 升级数据库版本 10 → 11
- 移除 `name` 字段的 `UNIQUE` 约束
- 修改 `insertStation` 和 `insertStations` 方法，使用普通 `insert` 替代 `insertWithOnConflict(CONFLICT_REPLACE)`
- 新增 `isStationNameExists()` 方法用于检查名称是否被其他记录使用

**修改文件 2**: `app/src/main/java/com/bh6aap/ic705Cter/ui/components/StationSettingsDialog.kt`
- 保存前检查名称是否被其他地面站使用
- 如果名称重复，显示错误提示 "地面站名称 'xxx' 已被使用，请使用其他名称"
- 避免应用崩溃，提升用户体验

### 4. 主界面地面站列表功能

**需求**: 用户希望能够快速切换已保存的地面站位置。

**解决方案**: 在主界面 GPS 栏添加长按手势，弹出地面站列表进行快速切换。

**新增文件**: `app/src/main/java/com/bh6aap/ic705Cter/ui/components/StationListDialog.kt`
- 地面站列表对话框组件
- 显示所有已保存的地面站
- 当前选中的地面站显示勾选标记和特殊背景色
- 支持点击切换默认地面站
- 支持删除地面站（带确认对话框）
- 使用 Flow 监听数据库变化，实时更新列表

**修改文件**: `app/src/main/java/com/bh6aap/ic705Cter/MainActivity.kt`
- 添加 `StationListDialog` 导入
- 修改 `GpsInfoBar` 函数，添加 `onLongPress` 参数
- 为 GPS 信息区域添加长按手势检测
- 添加 `showStationListDialog` 状态管理
- 长按 GPS 栏时弹出地面站列表
- 选择地面站后更新主界面显示的 GPS 信息

**使用方式**:
1. 在主界面长按 GPS 信息栏
2. 弹出地面站列表对话框
3. 点击要使用的地面站即可切换
4. 点击删除按钮可删除地面站

### 5. 地面站设置两种保存方案

**需求**: 用户希望地面站设置提供两种保存方式：覆盖默认和添加到库。

**解决方案**: 在地面站设置对话框中添加保存方式选择。

**修改文件**: `app/src/main/java/com/bh6aap/ic705Cter/ui/components/StationSettingsDialog.kt`

**新增功能**:

在海拔输入框下方添加"保存方式"选项：

**选项 1: 设为默认地面站**
- 替换当前默认地面站
- 用于卫星跟踪
- 清除其他地面站的默认标记
- 保存后提示 "已设为默认地面站"

**选项 2: 添加到地面站库**
- 保存到库中，不设置为默认
- 可在主界面长按 GPS 栏切换
- 方便管理多个常用位置
- 保存后
