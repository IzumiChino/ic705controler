1.3 音频处理 API 选择

对于 Android 12 及以上系统的低延迟音频处理，推荐使用Oboe API作为主要音频接口。Oboe 是 Google 开发的 C++ 封装容器，能够在 Android 8.1（API 级别 27）及更高版本上调用 AAudio，在更低版本上使用 OpenSL ES，提供了统一的低延迟音频接口。

音频捕获配置要点：

- 使用 Oboe 的低延迟模式：builder.setPerformanceMode (oboe::PerformanceMode::LowLatency)

- 请求独占模式以获得最佳延迟：builder.setSharingMode (oboe::SharingMode::Exclusive)

- 设置音频用途为 AAUDIO_USAGE_GAME 以优化延迟：builder.setUsage (oboe::Usage::Game)

- 使用 48000 Hz 采样率以避免采样率转换带来的延迟

根据实际测试，遵循这些配置建议可以将音频延迟控制在 20 毫秒以内，而不使用低延迟模式时延迟可达 205 毫秒。

二、核心功能技术实现方案

2.1 实时音频处理与 Morse 信号检测

音频流获取与预处理：使用 Oboe API 从麦克风获取 PCM 音频数据，采样率设置为 48000 Hz，音频格式为 16 位 PCM。音频预处理流程包括：

1. 降噪处理：使用数字信号处理技术去除环境噪声，提高 Morse 信号的信噪比

2. 带通滤波：设置 400-1200 Hz 的带通滤波器，这是 Morse 信号的典型频率范围

3. 自动增益控制（AGC）：实时调整音频增益，补偿信号强度变化

Morse 信号特征提取：核心算法基于快速傅里叶变换（FFT）检测特定频率的存在。实现步骤包括：

- 对音频数据执行 FFT 变换，将时域信号转换为频域

- 检测 400-1200 Hz 范围内的峰值频率

- 使用 Goertzel 算法进行精确的频率分析，该算法在特定频率检测上效率更高

2.2 频谱瀑布图技术实现

频谱瀑布图是本项目的核心视觉功能，需要实现实时频谱显示和时间轴滚动效果。

渲染技术选择：对于高频刷新率的动态频谱显示，建议使用 SurfaceView 或 TextureView 配合独立渲染线程，避免阻塞主线程造成界面卡顿。两者的主要区别在于：

- SurfaceView：在独立线程中渲染，适合需要高帧率的场景

- TextureView：将内容渲染到 OpenGL ES 纹理中，支持 View 层级中的变换操作

FFT 频谱计算：频谱分析使用快速傅里叶变换（FFT）算法，将计算复杂度从 O (N²) 降低到 O (N log N)。关键技术要点：

- FFT 窗口大小：建议使用 1024 或 2048 个采样点，这是 2 的幂次方，FFT 算法效率最高

- 加窗处理：使用汉宁窗或海明窗减少频谱泄漏

- 频谱更新频率：建议每秒 30-60 帧，平衡视觉效果和性能消耗

瀑布图实现原理：瀑布图本质上是一系列连续的频谱图按时间顺序排列。实现方案包括：

- 使用 OpenGL ES 2.0 进行高性能渲染，充分利用 GPU 加速

- 维护一个频谱数据缓冲区，保存最近 10 秒的频谱信息

- 实现平滑的滚动动画效果，新的频谱数据从底部进入，旧数据从顶部消失

- 使用颜色映射显示信号强度，如蓝→绿→黄→红的渐变方案

2.3 自动频率跟踪功能

自动频率跟踪是解码器的关键功能之一，能够实时检测和锁定 Morse 信号的频率。

频率检测算法：

- 使用 FFT 检测 400-1200 Hz 范围内的最强信号频率

- 实现自动频率检测器（AFD），自动将工作频率调整到最近的信号峰值

- 频率扫描精度：建议使用 50 Hz 的扫描步长，在精度和速度之间取得平衡

频率锁定机制：

- 用户可以通过点击瀑布图上的信号峰值来手动锁定频率

- 自动模式下，系统持续检测频率变化，当信号频率偏移超过一定阈值时自动调整

- 锁定状态下，解码器将频率固定，避免在信号不稳定时频繁调整

2.4 自动 WPM 跟踪技术

WPM（Words Per Minute）跟踪功能能够自动检测 Morse 码的发送速度，支持 4-50 WPM 的范围。

WPM 检测算法：

- 统计多个 "点" 的持续时间，取中位数作为单位时间长度

- 使用最小二乘法拟合算法，最小化 | t-δt|、|t-3δt|、|t-7δt | 的误差

- 其中 δt 是点的持续时间，3δt 是划的持续时间，7δt 是词间间隔

自适应跟踪机制：

- 解码器持续分析接收到的 Morse 信号的时间特征

- 在检测到 WPM 变化时，自动调整解码参数

- 支持快速跟踪，能够即时适应 WPM 速度变化

- 提供 WPM 锁定功能，用户可以固定解码速度

2.5 低延迟解码优化策略

实现 1 秒以内的低延迟解码是项目的技术难点，需要从多个层面进行优化。

音频延迟控制：

- 使用 Oboe 的低延迟模式，延迟可控制在 10 毫秒以内

- 配置合适的音频缓冲区大小，建议使用双缓冲区模式

- 避免在音频回调中进行复杂计算，将处理逻辑移到独立线程

处理延迟优化：

- 使用 C++ 实现核心信号处理算法，通过 NDK 调用提高执行效率

- 采用定点运算代替浮点运算，减少 CPU 负担

- 实现高效的内存管理，避免频繁的内存分配和释放

渲染延迟降低：

- 使用硬件加速渲染，充分利用 GPU 能力

- 实现高效的频谱数据更新机制，只重绘变化部分

- 使用缓存技术，避免重复计算相同的频谱数据

5.2 FFT 频谱分析实现

快速傅里叶变换算法：

public class FFTProcessor {

&#x20;   public double\[] performFFT(short\[] data) {

&#x20;       int n = data.length;

&#x20;       int m = (int) (Math.log(n) / Math.log(2));

&#x20;      &#x20;

&#x20;       // 转换为复数形式

&#x20;       Complex\[] x = new Complex\[n];

&#x20;       for (int i = 0; i < n; i++) {

&#x20;           x\[i] = new Complex(data\[i], 0);

&#x20;       }

&#x20;      &#x20;

&#x20;       // 执行FFT

&#x20;       fft(x, m);

&#x20;      &#x20;

&#x20;       // 计算幅度谱

&#x20;       double\[] magnitude = new double\[n/2];

&#x20;       for (int i = 0; i < n/2; i++) {

&#x20;           magnitude\[i] = x\[i].abs() / n;

&#x20;       }

&#x20;      &#x20;

&#x20;       return magnitude;

&#x20;   }

&#x20;  &#x20;

&#x20;   private void fft(Complex\[] x, int m) {

&#x20;       int n = x.length;

&#x20;       if (n == 1) return;

&#x20;      &#x20;

&#x20;       // 分治处理

&#x20;       Complex\[] even = new Complex\[n/2];

&#x20;       Complex\[] odd = new Complex\[n/2];

&#x20;       for (int i = 0; i < n/2; i++) {

&#x20;           even\[i] = x\[2\*i];

&#x20;           odd\[i] = x\[2\*i+1];

&#x20;       }

&#x20;      &#x20;

&#x20;       fft(even, m-1);

&#x20;       fft(odd, m-1);

&#x20;      &#x20;

&#x20;       // 合并结果

&#x20;       for (int k = 0; k < n/2; k++) {

&#x20;           Complex t = odd\[k].multiply(Complex.exp(-2 \* Math.PI \* k / n \* Complex.I));

&#x20;           x\[k] = even\[k].add(t);

&#x20;           x\[k + n/2] = even\[k].subtract(t);

&#x20;       }

5.3 Morse 解码算法

频率检测与 WPM 分析：

public class MorseDecoder {

&#x20;   public double detectFrequency(double\[] fftResult) {

&#x20;       // 在400-1200 Hz范围内寻找峰值

&#x20;       int startIndex = (int) (400.0 / 48000.0 \* fftResult.length);

&#x20;       int endIndex = (int) (1200.0 / 48000.0 \* fftResult.length);

&#x20;      &#x20;

&#x20;       double maxValue = 0;

&#x20;       int maxIndex = 0;

&#x20;       for (int i = startIndex; i < endIndex; i++) {

&#x20;           if (fftResult\[i] > maxValue) {

&#x20;               maxValue = fftResult\[i];

&#x20;               maxIndex = i;

&#x20;           }

&#x20;       }

&#x20;      &#x20;

&#x20;       return maxIndex \* 48000.0 / fftResult.length;

&#x20;   }

&#x20;  &#x20;

&#x20;   public double analyzeWPM(short\[] pcmData) {

&#x20;       // 检测信号的时间特征

&#x20;       List\<Double> pulseDurations = detectPulses(pcmData);

&#x20;      &#x20;

&#x20;       if (pulseDurations.isEmpty()) {

&#x20;           return 20.0; // 默认值

&#x20;       }

&#x20;      &#x20;

&#x20;       // 统计点和划的持续时间

&#x20;       List\<Double> dots = new ArrayList<>();

&#x20;       List\<Double> dashes = new ArrayList<>();

&#x20;      &#x20;

&#x20;       for (double duration : pulseDurations) {

&#x20;           if (duration < 0.1) { // 点

&#x20;               dots.add(duration);

&#x20;           } else { // 划

&#x20;               dashes.add(duration);

&#x20;           }

&#x20;       }

&#x20;      &#x20;

&#x20;       // 计算平均点持续时间

&#x20;       double avgDot = dots.stream().mapToDouble(Double::doubleValue).average().orElse(0.05);

&#x20;      &#x20;

&#x20;       // 计算WPM (WPM = 60 / (avgDot \* 1.2))

&#x20;       return 60.0 / (avgDot \* 1.2);

&#x20;   }

&#x20;  &#x20;

&#x20;   private List\<Double> detectPulses(short\[] pcmData) {

&#x20;       List\<Double> durations = new ArrayList<>();

&#x20;       boolean isSignal = false;

&#x20;       double startTime = 0;

&#x20;      &#x20;

&#x20;       for (int i = 0; i < pcmData.length; i++) {

&#x20;           short sample = pcmData\[i];

&#x20;           boolean aboveThreshold = Math.abs(sample) > 1000; // 信号阈值

&#x20;          &#x20;

&#x20;           if (aboveThreshold != isSignal) {

&#x20;               if (isSignal) {

&#x20;                   double duration = (i - startTime) / 48000.0;

&#x20;                   durations.add(duration);

&#x20;               }

&#x20;               isSignal = aboveThreshold;

&#x20;               startTime = i;

&#x20;           }

&#x20;       }

&#x20;      &#x20;

&#x20;       return durations;

&#x20;   }

}
